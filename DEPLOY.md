# Deployment guide

Production checklist for deploying this Laravel + Vue/Inertia app.

---

## 1. Server requirements

| Software | Minimum version | Notes |
|---|---|---|
| **PHP** | 8.0 | 8.2+ recommended (8.0 EOL'd Nov 2023) |
| **PostgreSQL** | **15+** | Migration `2026_04_29_120000` uses `NULLS NOT DISTINCT` (PG 15+) |
| **Node.js** | 18+ | For `npm run build` |
| **Composer** | 2.x | |
| **Web server** | Nginx (recommended) or Apache | |
| **systemd / cron** | any | Required for scheduled tasks (audit cleanup, future backups) |

PHP extensions needed: `pdo_pgsql`, `mbstring`, `xml`, `curl`, `zip`, `bcmath`, `tokenizer`, `openssl`, `intl`, `json`, `fileinfo`.

## 2. Initial deploy

### 2.1 Clone the repo
```bash
cd /var/www
git clone <your-repo-url> excel
cd excel
```

### 2.2 Install dependencies
```bash
composer install --no-dev --optimize-autoloader
npm ci
npm run build
```

### 2.3 Set up `.env`

Copy the example and fill in real values:
```bash
cp .env.example .env
nano .env
```

**Critical settings for production:**

```env
APP_NAME="Excel"
APP_ENV=production           # ← IMPORTANT: not 'local'
APP_DEBUG=false              # ← IMPORTANT: false in production!
APP_URL=https://your-domain.tld
APP_KEY=                     # generated by next command

LOG_CHANNEL=daily
LOG_DAILY_DAYS=14
LOG_LEVEL=info               # not 'debug' in prod

AUDIT_LOG_RETENTION_DAYS=90  # how long to keep audit log entries

DB_CONNECTION=pgsql
DB_HOST=127.0.0.1            # or your DB server
DB_PORT=5432
DB_DATABASE=excel_db
DB_USERNAME=excel_user
DB_PASSWORD=<strong-random-password>

SESSION_DRIVER=database      # 'file' breaks on multi-server; use 'database' or 'redis'
CACHE_DRIVER=database        # same — 'file' is single-server only
QUEUE_CONNECTION=sync        # 'sync' processes jobs inline; ok if no async work
SESSION_LIFETIME=120

# Mail is currently not used (forgot-password is disabled).
# If you re-enable email later — fill these:
MAIL_MAILER=smtp
MAIL_HOST=smtp.your-provider.com
MAIL_PORT=587
MAIL_USERNAME=
MAIL_PASSWORD=
MAIL_ENCRYPTION=tls
MAIL_FROM_ADDRESS=noreply@your-domain.tld
MAIL_FROM_NAME="${APP_NAME}"
```

### 2.4 Generate APP_KEY
```bash
php artisan key:generate
```
This writes a unique key into `.env`. **Do not** copy `APP_KEY` from dev — every environment must have its own.

### 2.5 Database setup
```bash
# Create database + user (one-time)
sudo -u postgres createdb excel_db
sudo -u postgres createuser excel_user --pwprompt

# Run migrations
php artisan migrate --force
```

### 2.6 Create the first admin
```bash
php artisan tinker
```
```php
$u = App\Models\User::create([
    'name' => 'Admin',
    'email' => 'admin@your-domain.tld',
    'password' => Hash::make('strong-temp-password'),
    'email_verified_at' => now(),
]);
App\Models\Sheet::makeUserAdmin($u);
```

### 2.7 File permissions
The web server user (often `www-data` or `nginx`) needs **write** access to:
```bash
sudo chown -R www-data:www-data storage bootstrap/cache
sudo chmod -R 775 storage bootstrap/cache
```

### 2.8 Cache configs for performance
```bash
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

### 2.9 System cron for scheduled tasks
Add this **one** line to crontab to enable audit-log retention cleanup and any future scheduled jobs:
```bash
crontab -e
```
Add:
```
* * * * * cd /var/www/excel && php artisan schedule:run >> /dev/null 2>&1
```

## 3. Nginx config (example)

```nginx
server {
    listen 80;
    server_name your-domain.tld;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.tld;
    root /var/www/excel/public;

    # SSL — set up via Let's Encrypt: `certbot --nginx -d your-domain.tld`
    ssl_certificate /etc/letsencrypt/live/your-domain.tld/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.tld/privkey.pem;

    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";
    add_header X-XSS-Protection "1; mode=block";
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

    index index.php;
    charset utf-8;

    # Big payloads for large XLSX imports
    client_max_body_size 50M;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location = /favicon.ico { access_log off; log_not_found off; }
    location = /robots.txt  { access_log off; log_not_found off; }

    error_page 404 /index.php;

    location ~ \.php$ {
        fastcgi_pass unix:/run/php/php8.2-fpm.sock;   # adjust PHP version
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
    }

    # Block access to dotfiles + sensitive paths
    location ~ /\.(?!well-known).* { deny all; }
    location ~ ^/(\.env|\.git|composer\.(json|lock)|package(-lock)?\.json|vendor|storage) { deny all; }
}
```

Reload Nginx:
```bash
sudo nginx -t && sudo systemctl reload nginx
```

### HTTPS via Let's Encrypt
```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.tld
```

## 4. Subsequent deploys (update)

```bash
cd /var/www/excel
git pull origin main
composer install --no-dev --optimize-autoloader
npm ci
npm run build
php artisan migrate --force
php artisan config:cache
php artisan route:cache
php artisan view:cache
sudo systemctl reload php8.2-fpm   # let PHP pick up changes
```

You can save the above as `deploy.sh` in the project root.

## 5. Post-deploy verification

| Check | Command / URL |
|---|---|
| Site loads over HTTPS | `https://your-domain.tld` |
| Login works | log in as the admin you created |
| Migrations applied | `php artisan migrate:status` |
| Audit cleanup scheduled | `php artisan schedule:list` shows `audit-log:cleanup` |
| Logs writing | `tail -f storage/logs/laravel-$(date +%Y-%m-%d).log` |
| Static assets serve | DevTools → Network → no 404 on `/build/...` files |

## 6. What's NOT in this guide (still TODO before going live)

These are operational concerns the deploy script can't solve. Track them separately:

1. **Database backups.** Set up `pg_dump` to S3/Backblaze on a daily cron, or use the managed-PG provider's snapshots.
2. **Off-site backup verification.** Periodically test `pg_restore` from a backup — bullets that are never test-fired don't fire.
3. **Monitoring / alerting.** Sentry / Bugsnag / similar for error tracking; UptimeRobot or similar for uptime; disk/CPU alerts.
4. **Two-factor authentication.** The app has no 2FA — admin password leak = full takeover.
5. **`/health` endpoint.** Not implemented; recommended for load balancers / uptime checks.
6. **Last-admin-deletion guard.** `UserController::destroy` blocks deleting yourself but not the last admin.
7. **PHP / Laravel upgrade.** PHP 8.0 and Laravel 9 are EOL. Plan to upgrade to PHP 8.2+ and Laravel 10/11.

## 7. Account recovery (if you lose admin access)

If admin password is lost and email recovery is not configured (it's currently disabled by design):

```bash
# SSH to the server
ssh you@your-server
cd /var/www/excel
php artisan tinker
```

```php
$u = App\Models\User::where('email', 'admin@your-domain.tld')->first();
$u->password = Hash::make('NewTempPassword123');
$u->save();
// Then log in and change it via the profile UI.
```

This is the **only** recovery path — guard your SSH keys carefully.

## 8. Security checklist before public launch

- [ ] `APP_ENV=production`, `APP_DEBUG=false` in `.env`
- [ ] Unique `APP_KEY` generated on the server (not copied from dev)
- [ ] HTTPS enforced (HTTP redirects to HTTPS)
- [ ] At least 2 admin accounts (mutual recovery)
- [ ] Database password is long and random
- [ ] `storage/` and `bootstrap/cache/` writable by web user only
- [ ] `.env` file not readable by anyone except the web user (`chmod 600 .env`)
- [ ] Daily DB backups running (and tested!)
- [ ] Cron `schedule:run` line added (verifies with `php artisan schedule:list`)
- [ ] SSH access locked down (key-only, fail2ban, no root login)
